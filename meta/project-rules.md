# Project Rules для AI-агента

Стабильные правила, которые агент всегда применяет при работе с `system-design-db`.

## Принципы

- **Think before writing:** явно назвать предположения, неоднозначности и tradeoffs.
- **Single source of truth:** факты об одной сущности живут на одной canonical странице.
- **Surgical changes:** менять только то, что связано с задачей.
- **Evidence-driven execution:** после изменений запускать проверки и фиксировать результат.

## Рабочий цикл

1. Уточнить цель и scope.
2. Сделать короткий план.
3. Выполнить маленькими порциями.
4. Проверить: `python3 meta/scripts/validate.sh`.
5. Зафиксировать след: обновить overview, decisions или audit log при необходимости.

## Стоп-критерии

- Успех: критерии выполнены, проверки проходят.
- Блокировка: не хватает данных, доступа или решения человека.
- Unsafe: действие нарушает policy.
- Repeated failure: одинаковая ошибка повторилась больше двух раз.

## Специфика system-design-db

- Это Markdown vault + Git-репозиторий.
- Каждая заметка — один файл с устойчивой темой.
- Правила именования: `kebab-case`, английские названия, `OVERVIEW.md` для обзорных страниц.
- Связи между заметками — Markdown-ссылки на существующие файлы.
- Не оставлять битые ссылки как заглушки.
