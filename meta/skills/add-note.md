# Skill: add-note

Нужно добавить новую концепцию, паттерн или практическую заметку в `system-design-db`.

## Алгоритм

1. Определи canonical дом: `patterns/`, `tools/` или `sources/`.
2. Проверь, нет ли уже похожей страницы через `rg`.
3. Создай файл в `kebab-case`.
4. Добавь ссылку в ближайший `OVERVIEW.md`.
5. Если заметка ссылается на существующую сущность, используй Markdown-ссылку на canonical страницу.
6. Запусти `python3 meta/scripts/validate.sh`.

## Проверка результата

- Нет битых ссылок.
- Нет дублей.
- Overview обновлен.
- Новая заметка имеет понятную роль.
