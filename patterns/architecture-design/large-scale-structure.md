# Large-Scale Structure

## Проблема

Когда система состоит из десятков Bounded Contexts, разработчикам трудно ориентироваться в общей картине. Без «карты» каждый видит только свой кусок, дублирование растёт, интеграции ломаются.

## Решение

Large-Scale Structure — это высокоуровневый взгляд на систему: метафоры, слои ответственности, evolving order. Структура помогает понимать роль каждого компонента, не углубляясь в детали.

## Паттерны Large-Scale Structure

### System Metaphor
Метафора, понятная всем участникам. Например: «Наша система как почтовое отделение — сообщения поступают, сортируются по адресатам, доставляются в ящики». Не обязана быть точной — обязана быть полезной для коммуникации.

### Evolving Order
Структура не навязывается сверху раз и навсегда. Она проявляется по мере развития системы и регулярно пересматривается. Навязывание жёсткой структуры на ранних этапах душит дизайн.

### Responsibility Layers
Система делится на слои ответственности (не путать с техническими слоями layered architecture):
- **Operations:** core бизнес-логика.
- **Potential:** аналитика, прогнозирование, ML.
- **Decision Support:** отчёты, дашборды, BI.
- **Capability:** инфраструктурные сервисы (auth, хранение, нотификации).

Каждый BC может жить в определённом слое, что упрощает понимание его роли.

### Pluggable Component Framework
Интерфейсы между компонентами стандартизированы так, что компоненты можно заменять. Каждый BC — заменяемый модуль с чётким контрактом.

## Когда применять

- Система выросла настолько, что новички теряются.
- Разные команды дублируют функциональность, не зная о существовании аналогов.
- Нужна общая road map для архитектурных решений.

## Когда НЕ применять

- Система маленькая (меньше 5 BC) — overhead не оправдан.
- Команда ещё не нашла стабильную структуру — навязывание убьёт эволюцию.

## Tradeoffs

Плюсы: общее понимание, снижение дублирования, упрощение кросс-командной коммуникации.
Минусы: риск навязывания преждевременной структуры, соблазн «подогнать» дизайн под метафору, а не под домен.
