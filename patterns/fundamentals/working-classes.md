# Working classes

## Проблема

Классы часто становятся dumping ground для несвязанной функциональности. Без дисциплины абстракция размывается, public interface раздувается, и класс перестаёт выполнять одну понятную роль.

## Решение

McConnell формулирует правила создания качественных классов: каждый класс должен быть **coherent abstraction with minimal public interface**.

## Критерии качества класса

### Хорошая абстракция

- Класс представляет одну концепцию предметной области.
- Интерфейс класса отражает эту концепцию, а не детали реализации.
- Из названия класса понятно, что он делает — без чтения кода.

### Минимальный public interface

- Public methods — только те, которые нужны клиентам класса.
- Всё остальное — private/protected.
- McConnell: «Если сомневаешься, делай private. Сделать public позже — один diff; сделать private — ломающий change».

### High cohesion

- Все методы класса используют большую часть полей класса.
- Если группа методов работает с подмножеством полей — это кандидат на выделение в отдельный класс.

### Low coupling

- Класс знает минимум о других классах.
- Зависимости injected, а не создаются внутри.
- Класс не зависит от internal details соседей.

## Принципы проектирования классов (по McConnell)

1. **Single Responsibility Principle** — одна причина для изменения.
2. **Interface Segregation** — клиенты не должны зависеть от методов, которые они не используют.
3. **Open/Closed** — открыт для расширения, закрыт для модификации.
4. **Liskov Substitution** — наследники должны быть заменяемы без нарушения контракта.
5. **Dependency Inversion** — завись от абстракций, не от конкретных реализаций.

## Когда не нужен отдельный класс

- Класс с одним методом (кроме стратегий/команд) — вероятно, функция.
- Класс без состояния — вероятно, namespace.
- Класс, который только делегирует — вероятно, unnecessary indirection.

## Антипаттерны

- **God class** — знает слишком много, делает слишком много.
- **Data class** — только геттеры/сеттеры без поведения; логика размазана по другим классам.
- **Poltergeist** — класс, который только вызывает другие классы без собственной логики.
