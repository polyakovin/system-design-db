# «Возьмите инициативу на себя»: готовимся к System Design Interview

Статья из блога AvitoTech на Habr — опыт компании Avito в проведении секции System Design на собеседованиях. Практические рекомендации для кандидатов и описание подхода Avito к проектированию.

## Описание

- Автор: Артём Ермилов (фронтенд-инженер, Avito).
- Опубликовано: ~2023.
- Формат: 5 мин чтения, 41K просмотров.
- Темы: System Design Interview, frontend-проектирование, архитектура, Avito hiring.

## Ключевые темы

- **Зачем Avito проводит System Design секцию:** проверка навыков проектирования, умения взвешивать trade-offs, выявлять риски.
- **Кросс-функциональные команды:** команды Avito сами проектируют и реализуют архитектуру (бэкенд, фронтенд, мобильные разработчики, QA).
- **3 этапа интервью:** сбор требований (5-10 мин) → high-level дизайн (20 мин) → deep dive (30 мин).
- **Клиент-серверная архитектура:** обсуждение системы целиком — клиенты, балансеры, BFF, серверы приложений, кэши, очереди, хранилища.
- **Полезные ресурсы:** Alex Xu, Martin Kleppmann, Educative, C4 Model, YouTube-каналы ByteByteGo и Front-End Engineer.

## Практическая ценность

- Инсайдерский взгляд на то, как крупная российская tech-компания оценивает архитектурные навыки.
- Конкретные примеры вопросов для deep dive (технологический стек, клиент-серверное взаимодействие, метрики, тестирование).
- Связь с экосистемой System Design Interview (книги Alex Xu и Kleppmann).
