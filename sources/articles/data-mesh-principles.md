# Data Mesh Principles and Logical Architecture

Original article by Zhamak Dehghani (Thoughtworks), published on martinfowler.com in December 2020. Defines the four foundational principles of data mesh: domain-oriented decentralized data ownership, data as a product, self-serve data infrastructure as a platform, and federated computational governance. Establishes the high-level logical architecture that bridges operational and analytical data planes under domain-aligned boundaries. This is the canonical reference for the data mesh paradigm.

## Key Topics

- Domain-oriented decentralized data ownership and architecture
- Data as a product
- Self-serve data infrastructure as a platform
- Federated computational governance
- Bridging operational and analytical data planes
- Logical architecture of a data mesh
- Comparison with monolithic data lake/warehouse approaches
