# System Design: Общие принципы прохождения интервью по проектированию ИТ-систем

Фрагмент книги издательства «Питер» по подготовке к System Design Interview. Опубликован на Habr как промо-материал. Содержит пошаговую методику прохождения интервью по проектированию ИТ-систем.

## Описание

- Автор: издательство «Питер» (перевод книги Alex Xu — System Design Interview).
- Опубликовано: ~2023.
- Формат: отрывок из книги, 9 мин чтения.
- Темы: System Design Interview, архитектурное мышление, сбор требований, high-level design, deep dive.

## Ключевые темы

- **4-шаговая методика интервью:** понять задачу → high-level дизайн → deep dive → подведение итогов.
- **Сбор требований:** функциональные и нефункциональные требования, масштаб, аудитория, ограничения.
- **High-level дизайн:** блок-схемы, ключевые компоненты (клиенты, API, серверы, хранилища, кэши, CDN, очереди).
- **Приблизительные расчёты (back-of-the-envelope):** оценка нагрузки, matching масштабу.
- **Антипаттерны:** спешка с решением, переусложнение, узкий кругозор.

## Практическая ценность

- Структурированная методика для подготовки к System Design Interview.
- Подкрепляет подход из книг Alex Xu (уже добавленных в vault).
- Акцент на процессе, а не на «правильном ответе».
