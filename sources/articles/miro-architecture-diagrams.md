# Mapping Your Software: Your Guide to Essential Architecture Diagrams

Статья из блога Miro — обзор типов архитектурных диаграмм и их применения в процессе разработки ПО. Полезна как визуальный гайд по диаграммам и роли Miro как инструмента для совместного моделирования.

## Описание

- Автор: Kat Boogaard (Contributing Writer, Miro).
- Опубликовано: 2024-05-21.
- Категории диаграмм: Structure (Component, Class), Behavior (Sequence, Activity, State, Use Case, Data Flow), Deployment.
- Инструмент: Miro — collaborative diagramming с шаблонами архитектурных диаграмм.

## Ключевые темы

- **Software architecture diagram** — единый источник истины о структуре системы: компоненты, связи, внешние системы, интерфейсы, границы.
- **C4 model** — упоминается в контексте визуализации (C4 Model как подход).
- **Применение диаграмм** на всех этапах: планирование → разработка → тестирование → документация → ретроспектива.
- **8 типов диаграмм** с шаблонами: Component, Class, Sequence, Activity, State, Use Case, Data Flow, Deployment.

## Практическая ценность

- Обзорная карта типов архитектурных диаграмм — что и когда применять.
- Подтверждает важность диаграмм как инструмента коммуникации с не-техническими стейкхолдерами.
- Связь с C4 Model и практиками документирования архитектуры.
