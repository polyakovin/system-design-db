# A Philosophy of Software Design

Конспект книги John Ousterhout «A Philosophy of Software Design» — про управление сложностью через глубокие модули, стратегический подход к проектированию и борьбу с эрозией архитектуры.

## Описание

- Снижение сложности как главная цель софтверного дизайна
- Глубокие модули (deep modules): много функциональности за простым интерфейсом
- Разница между стратегическим и тактическим программированием
- Инкапсуляция, слои абстракции, работа с ошибками
- Комментарии и документация как часть дизайна
- Практическая ценность: прямые применимые принципы для code review и архитектурных решений
- Связанные разделы vault: `system-design-principles`, `api-design`
